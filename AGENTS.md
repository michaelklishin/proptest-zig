# Instructions for AI Agents

## What is This Codebase?

A Zig port of [Rust `proptest`](https://github.com/proptest-rs/proptest): a small,
Hypothesis-style property-based testing library with integrated shrinking.


## Target Zig Version(s)

 * Target Zig 0.16.0


## Build and Test

```bash
zig build

zig build test
```

Tests do not require any external service. They are split into two layers:

 * Unit tests live next to the implementation as `test "..."` blocks
 * Test files in `tests/` exercise the public API end-to-end


## Repository Layout

 * `src/root.zig`: public API, re-exports
 * `src/runner.zig`: `Runner`, `Config`, the `check` driver, RNG seeding
 * `src/strategy.zig`: combinators that work for any value type: `Just`,
   `Map`, `Filter`, `OneOf`
 * `src/num.zig`: integer and float strategies, plus their value trees
 * `src/bool.zig`: `bool` and weighted-bool strategies
 * `src/collection.zig`: slice, byte string, and ASCII string strategies
 * `src/tuple.zig`: heterogeneous-tuple combinator
 * `tests/`: integration-style tests against the public API

The two core contracts (described in detail in `src/root.zig`):

 * A **strategy** is any value with `pub const Value`, a `Tree` type alias,
   and `pub fn newTree(self, runner, allocator) !Tree`.
 * A **value tree** carries the current generated value plus a binary-search
   shrink protocol: `current()`, `simplify()`, `complicate()`, optional
   `deinit()`.

Zig has no traits, so the two contracts are duck-typed via comptime.


## Code Style

 * `camelCase` for functions and locals, `PascalCase` for types,
   `snake_case` for file names
 * Accept `std.mem.Allocator` parameters; never use a global allocator
 * Use `errdefer` for cleanup on the error path
 * Prefer `std.Random` over thread-local RNG: strategies must be
   reproducible from a seed


## Tests

 * Tests should be descriptive and easy to read
 * Use `std.testing.expectEqual`, `expectEqualStrings`,
   `expectEqualSlices`, `expect`, `expectError`
 * For statistical assertions (e.g. `OneOf` weight distribution),
   pin the seed: never assert on unseeded RNG output
 * Property-tests of the property tester (meta tests) live in
   `tests/meta_test.zig`: they're the highest-leverage tests because they
   exercise the shrinker on its own primitives


## Comments

 * Only add comments that express the non-obvious *why*, both in tests and
   in the implementation
 * Keep them short
 * Pay attention to grammar: full stops, articles, capitalisation


## Source of Domain Knowledge

 * [Rust `proptest`](https://github.com/proptest-rs/proptest): the reference port
 * [Hypothesis](https://hypothesis.readthedocs.io/): for shrinking ideas
 * [Hedgehog](https://hedgehog.qa/): for integrated shrinking semantics

Treat the Rust proptest source as the first-party source of truth when
porting a feature.

## Style Guide

 * Never add full stops to Markdown list items


## Git Instructions

 * Do not commit changes automatically without explicit permission
 * Never add yourself to the list of commit co-authors
 * Never mention yourself in commit messages in any way (no
   "Generated by", no AI tool links)


## After Completing a Task

After completing a task, look very carefully for gaps in the implementation,
test coverage, missed opportunities to use idiomatic Zig patterns,
correctness bugs in the shrinker, and ways to lean on the Zig type system
for additional safety.

"There's nothing meaningful to improve" is an acceptable answer.
